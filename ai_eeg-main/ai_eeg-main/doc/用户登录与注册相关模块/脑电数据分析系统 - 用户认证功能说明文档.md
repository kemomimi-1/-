# 脑电数据分析系统 - 用户认证功能说明文档

## 1. 系统概述

本系统是基于OpenBCI_GUI v6.0.0 beta.1版本的脑电数据分析平台，集成AI大模型和MCP服务功能，用于分析处理三种类型的脑电数据流：TimeSeriesRaw（原始时间序列）、TimeSeriesFilt（滤波时间序列）和AvgBandPower（平均频带功率）。系统采用Spring Boot框架构建，使用InfluxDB 3.2.1作为时序数据库，通过UDP协议接收脑电数据。

## 2. 系统架构组件

### 2.1 Web配置层（WebConfig.java）

**类职责：** 负责Web应用的全局配置，包括静态资源管理、跨域设置和视图控制。

#### 核心方法功能：

**`addResourceHandlers(ResourceHandlerRegistry registry)`**

- **功能：** 配置静态资源文件的访问路径映射
- **实现逻辑：** 将所有HTTP请求路径映射到`classpath:/static/`目录
- **缓存策略：** 设置3600秒（1小时）的缓存期，提升前端资源加载性能
- **使用意义：** 支持前端页面、脚本、样式表等静态资源的高效访问

```java
@Override
public void addResourceHandlers(ResourceHandlerRegistry registry) {
    // 配置静态资源路径映射
    registry.addResourceHandler("/**")
            .addResourceLocations("classpath:/static/")
            .setCachePeriod(3600); // 缓存1小时
}
```

**`addViewControllers(ViewControllerRegistry registry)`**

- **功能：** 配置默认页面路由
- **实现逻辑：** 将根路径"/"重定向到index.html页面
- **使用意义：** 确保用户访问根域名时能正确加载主页面

```java
@Override
public void addViewControllers(ViewControllerRegistry registry) {
    // 根路径重定向到index.html
    registry.addViewController("/").setViewName("forward:/index.html");
}
```

**`addCorsMappings(CorsRegistry registry)`**

- **功能：** 配置跨域资源共享（CORS）规则
- **覆盖范围：** 所有以`/api/`开头的接口
- **允许的操作：** GET、POST、PUT、DELETE、OPTIONS请求方法
- **安全配置：** 支持凭证传递，允许所有来源和请求头
- **使用意义：** 支持前后端分离架构，解决浏览器跨域限制问题

```java
@Override
public void addCorsMappings(CorsRegistry registry) {
    registry.addMapping("/api/**")
            .allowedOriginPatterns("*")
            .allowedMethods("GET", "POST", "PUT", "DELETE", "OPTIONS")
            .allowedHeaders("*")
            .allowCredentials(true)
            .maxAge(3600);
}
```

### 2.2 认证控制层（AuthController.java）

**类职责：** 处理用户认证相关的HTTP请求，提供注册、登录、登出和状态查询功能。

#### 核心接口功能：

**`POST /api/auth/register`** - 用户注册接口

- 输入验证：
  - 用户名：6-20位字母数字组合（正则：`^[a-zA-Z0-9]{6,20}$`）
  - 密码：8-30位，必须包含至少一个字母和一个数字（正则：`^(?=.*[A-Za-z])(?=.*\\d)[A-Za-z\\d@$!%*?&]{8,30}$`）
- **业务逻辑：** 调用UserService进行用户创建
- **响应数据：** 返回用户ID、用户名和成功消息
- **异常处理：** 捕获业务异常并返回友好错误信息
- **使用意义：** 为新用户提供安全的账户创建功能，确保数据完整性

```java
@PostMapping("/register")
public ResponseEntity<?> register(@RequestBody RegisterRequest request) {
    try {
        // 验证用户名格式：6-20位字母数字
        if (!request.getUsername().matches("^[a-zA-Z0-9]{6,20}$")) {
            return ResponseEntity.badRequest()
                    .body(Map.of("error", "用户名必须是6-20位字母和数字组合"));
        }

        // 验证密码格式：8-30位，包含字母、数字
        if (!request.getPassword().matches("^(?=.*[A-Za-z])(?=.*\\d)[A-Za-z\\d@$!%*?&]{8,30}$")) {
            return ResponseEntity.badRequest()
                    .body(Map.of("error", "密码必须8-30位，包含至少一个字母和一个数字"));
        }

        User user = userService.register(request.getUsername(), request.getPassword());

        return ResponseEntity.ok(Map.of(
                "message", "注册成功",
                "userId", user.getId(),
                "username", user.getUsername()
        ));

    } catch (RuntimeException e) {
        return ResponseEntity.badRequest().body(Map.of("error", e.getMessage()));
    }
}
```

**`POST /api/auth/login`** - 用户登录接口

- **认证流程：** 验证用户名和密码的正确性
- **会话管理：** 登录成功后将用户信息存储到HttpSession中
- **会话数据：** 存储userId和username用于后续请求验证
- **响应数据：** 返回用户基本信息和登录状态
- **使用意义：** 建立用户会话，为后续脑电数据操作提供身份验证基础

```java
@PostMapping("/login")
public ResponseEntity<?> login(@RequestBody LoginRequest request, HttpSession session) {
    try {
        User user = userService.login(request.getUsername(), request.getPassword());

        // 设置会话
        session.setAttribute("userId", user.getId());
        session.setAttribute("username", user.getUsername());

        return ResponseEntity.ok(Map.of(
                "message", "登录成功",
                "userId", user.getId(),
                "username", user.getUsername()
        ));

    } catch (RuntimeException e) {
        return ResponseEntity.badRequest().body(Map.of("error", e.getMessage()));
    }
}
```

**`POST /api/auth/logout`** - 用户登出接口

- **会话清理：** 调用`session.invalidate()`销毁当前会话
- **资源释放：** 清除服务器端会话数据，释放内存资源
- **使用意义：** 确保用户安全退出，防止会话泄露

```java
@PostMapping("/logout")
public ResponseEntity<?> logout(HttpSession session) {
    session.invalidate();
    return ResponseEntity.ok(Map.of("message", "已退出登录"));
}
```

**`GET /api/auth/status`** - 认证状态查询接口

- **状态检测：** 检查当前会话中是否存在有效的用户信息
- 响应逻辑：
  - 已认证：返回用户ID、用户名和认证状态
  - 未认证：返回未认证标识
- **使用意义：** 支持前端页面的登录状态检查和用户信息显示

```java
@GetMapping("/status")
public ResponseEntity<?> getAuthStatus(HttpSession session) {
    Long userId = (Long) session.getAttribute("userId");
    String username = (String) session.getAttribute("username");

    if (userId != null) {
        return ResponseEntity.ok(Map.of(
                "authenticated", true,
                "userId", userId,
                "username", username
        ));
    } else {
        return ResponseEntity.ok(Map.of("authenticated", false));
    }
}
```

### 2.3 用户实体层（User.java）

**类职责：** 定义用户数据模型，映射数据库表结构，支持脑电数据传输的端口管理。

#### 核心字段功能：

**基础字段：**

- `id`：主键，自增长策略
- `username`：唯一用户名，最长20字符
- `password`：加密存储的用户密码

**脑电数据相关字段：**

- `rawPort`：原始脑电数据（TimeSeriesRaw）的UDP端口号
- `filtPort`：滤波脑电数据（TimeSeriesFilt）的UDP端口号
- `bandPort`：频带功率数据（AvgBandPower）的UDP端口号
- **使用意义：** 为每个用户分配独立的数据接收端口，实现多用户并发的脑电数据传输

```java
@Data
@Entity
@Table(name = "users")
@NoArgsConstructor
@AllArgsConstructor
public class User {
    @Id
    @GeneratedValue(strategy = GenerationType.IDENTITY)
    private Long id;

    @Column(unique = true, nullable = false, length = 20)
    private String username;

    @Column(nullable = false)
    private String password; // 加密存储

    @Column(name = "raw_port")
    private Integer rawPort;    // TimeSeriesRaw数据端口

    @Column(name = "filt_port")
    private Integer filtPort;   // TimeSeriesFilt数据端口

    @Column(name = "band_port")
    private Integer bandPort;   // AvgBandPower数据端口

    @Column(name = "is_active")
    private Boolean isActive = false;

    @Column(name = "last_connection")
    private LocalDateTime lastConnection;

    @CreationTimestamp
    @Column(name = "created_at")
    private LocalDateTime createdAt;

    @UpdateTimestamp
    @Column(name = "updated_at")
    private LocalDateTime updatedAt;
}
```

### 2.5 业务服务层（UserService.java）

**类职责：** 实现用户管理的核心业务逻辑，包括注册、登录、端口分配和状态管理。

#### 核心方法功能：

**`register(String username, String password)`**

- 业务流程：
  1. 检查用户名是否已存在
  2. 使用BCrypt算法加密密码
  3. 创建用户实体并持久化
  4. 记录注册日志
- **安全措施：** 密码采用BCrypt单向加密，无法反向解密
- **事务管理：** 使用@Transactional确保数据一致性
- **使用意义：** 提供安全可靠的用户注册功能

```java
@Transactional
public User register(String username, String password) {
    // 检查用户名是否已存在
    if (userRepository.findByUsername(username).isPresent()) {
        throw new RuntimeException("用户名已存在");
    }

    // 创建新用户
    User user = new User();
    user.setUsername(username);
    user.setPassword(passwordEncoder.encode(password));
    user.setIsActive(false);

    User savedUser = userRepository.save(user);
    log.info("新用户注册成功: {}", username);
    return savedUser;
}
```

**`login(String username, String password)`**

- 认证流程：
  1. 根据用户名查询用户信息
  2. 使用BCrypt验证密码正确性
  3. 更新最后连接时间
  4. 记录登录日志
- **密码验证：** 使用`passwordEncoder.matches()`进行安全比对
- **时间更新：** 自动记录用户最后活动时间
- **使用意义：** 提供安全的身份验证和用户活动追踪

```java
public User login(String username, String password) {
    Optional<User> userOpt = userRepository.findByUsername(username);

    if (userOpt.isEmpty()) {
        throw new RuntimeException("用户不存在");
    }

    User user = userOpt.get();
    if (!passwordEncoder.matches(password, user.getPassword())) {
        throw new RuntimeException("密码错误");
    }

    // 更新最后连接时间
    user.setLastConnection(LocalDateTime.now());
    userRepository.save(user);

    log.info("用户登录成功: {}", username);
    return user;
}
```

**`updateUserPorts(Long userId, int rawPort, int filtPort, int bandPort)`**

- **端口分配：** 为用户分配三个不同的UDP端口号
- **状态更新：** 将用户状态设置为活跃（isActive = true）
- 数据映射：
  - rawPort：接收TimeSeriesRaw原始脑电数据
  - filtPort：接收TimeSeriesFilt滤波处理后的数据
  - bandPort：接收AvgBandPower频带功率数据
- **使用意义：** 支持OpenBCI_GUI多数据流的并发传输，实现用户专属的数据通道

```java
@Transactional
public void updateUserPorts(Long userId, int rawPort, int filtPort, int bandPort) {
    User user = userRepository.findById(userId)
            .orElseThrow(() -> new RuntimeException("用户不存在"));

    user.setRawPort(rawPort);
    user.setFiltPort(filtPort);
    user.setBandPort(bandPort);
    user.setIsActive(true);

    userRepository.save(user);
    log.info("用户 {} 端口信息已更新: Raw={}, Filt={}, Band={}",
            userId, rawPort, filtPort, bandPort);
}
```

**`setUserInactive(Long userId)`**

- **状态管理：** 将用户状态设置为非活跃
- **资源释放：** 标记用户停止数据传输，便于端口资源回收

**`findById(Long userId)`**

- **查询功能：** 根据用户ID获取完整用户信息

### 2.6 WebSocket通知服务（WebSocketNotificationService.java）

**类职责：** 管理WebSocket连接，提供实时通知功能，支持脑电数据处理结果的即时推送。

- **断连记录：** 记录用户断开连接日志

**`notifyUser(Long userId, String type, Object data)`**

- **消息构建：** 构造包含类型、数据和时间戳的JSON消息
- **推送逻辑：** 向指定用户的WebSocket连接发送实时消息
- **连接检查：** 验证会话有效性，避免向无效连接发送消息
- **异常处理：** 捕获IO异常并记录错误日志
- 使用场景：
  - AI分析结果推送
  - 数据处理状态通知
  - 系统状态变更提醒
- **使用意义：** 实现脑电数据分析结果的实时反馈，提升用户体验

```java
public void notifyUser(Long userId, String type, Object data) {
    WebSocketSession session = userSessions.get(userId);
    if (session != null && session.isOpen()) {
        try {
            Map<String, Object> message = Map.of(
                    "type", type,
                    "data", data != null ? data : Map.of(),
                    "timestamp", System.currentTimeMillis()
            );

            String json = objectMapper.writeValueAsString(message);
            session.sendMessage(new TextMessage(json));

        } catch (IOException e) {
            log.error("发送WebSocket消息失败 - 用户: {}", userId, e);
        }
    }
}
```

## 3. 系统安全特性

### 3.1 密码安全

- 使用BCrypt算法进行密码哈希，具有动态盐值
- 支持密码强度验证，要求字母数字组合
- 密码单向加密，无法通过数据库反推原始密码

```java
private final BCryptPasswordEncoder passwordEncoder = new BCryptPasswordEncoder();

// 注册时加密密码
user.setPassword(passwordEncoder.encode(password));

// 登录时验证密码
if (!passwordEncoder.matches(password, user.getPassword())) {
    throw new RuntimeException("密码错误");
}
```

- 基于HttpSession的服务器端会话管理
- 自动会话超时和清理机制
- 登出时完整的会话销毁

```java
// 登录时设置会话
session.setAttribute("userId", user.getId());
```

### 3.3 输入验证
